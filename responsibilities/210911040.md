### What I Did

#### 1. Methodological Justification: Why Logistic Regression First

Before training, I verified the geometric alignment between the algorithm and the data.

**Linear Separability Check:** I produced a 2D scatter plot using the two features most correlated with the target (`worst concave points` and `worst perimeter`). The plot confirms that Malignant and Benign samples are largely separable by a linear boundary, which directly justifies using a linear classifier as the baseline.

**Why not Linear Regression?** Linear regression is designed for continuous outputs. Applying it to a binary classification problem produces predictions outside [0, 1], and its MSE loss is non-convex under sigmoid — it does not form a principled probabilistic model. Logistic Regression with Binary Cross-Entropy is the correct formulation for binary classification.

**Output:** `outputs/figures/step2/linear_separability_check.png`

#### 2. Model Training

Trained a Logistic Regression model using gradient descent optimization (`solver='saga'`, `max_iter=1000`) with a fixed random seed for reproducibility. The model uses the Sigmoid function to map the linear combination of features to a probability:

$$\hat{y} = \sigma(\mathbf{w}^\top \mathbf{x} + b) = \frac{1}{1 + e^{-(\mathbf{w}^\top \mathbf{x} + b)}}$$

Trained exclusively on `X_train_scaled` using the shared stratified splits from Step 1.

#### 3. Why Accuracy Alone Is Insufficient

Given the 63/37 class imbalance, a naive classifier that always predicts "Benign" achieves 62.7% accuracy without learning anything. I therefore prioritized **Recall** and **F1-Score** as primary evaluation metrics:

- **Recall (Sensitivity):** The fraction of actual Malignant cases correctly identified. A False Negative (predicting Benign when the tumor is Malignant) is the most dangerous clinical error.
- **F1-Score:** The harmonic mean of Precision and Recall — balances both types of error.

This framing establishes the evaluation standard used consistently across all models in the project.

### Technical Decisions and Justifications

| Decision | Justification |
|---|---|
| `solver='saga'` | Supports L1/L2 penalties and scales well; suitable for medium-sized datasets |
| `max_iter=1000` | Ensures convergence; default 100 was insufficient for this feature dimensionality |
| Recall as primary metric | Medical context: False Negatives (missed cancer) are more dangerous than False Positives |
| Same data splits as Step 1 | Fair comparison: all models trained and evaluated on identical partitions |
