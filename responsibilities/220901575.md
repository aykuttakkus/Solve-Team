### What I Did

#### 1. Experimental Design Principles

To isolate the effect of each regularization technique, I maintained a controlled experiment structure:

- **Same architecture** as Step 3 (30→64→32→1, ReLU, Sigmoid output)
- **Same dataset splits** from Step 1
- **Same optimizer** (Adam) and **same maximum epochs**
- **Only one variable changed** per experiment (the regularization method)

This design allows direct attribution of performance changes to the regularization technique rather than confounding variables.

#### 2. MLP + L2 Regularization (Week 4, Part 2)

**Theoretical basis:** L2 regularization adds a penalty term to the loss function:

$$\tilde{J}(\mathbf{w}) = J(\mathbf{w}) + \frac{\alpha}{2}\mathbf{w}^\top\mathbf{w}$$

This modifies the weight update rule to include a shrinkage factor:

$$\mathbf{w} \leftarrow (1 - \epsilon\alpha)\mathbf{w} - \epsilon\nabla_\mathbf{w} J$$

**Connection to the dataset (from Step 1):** The multicollinearity analysis in Step 1 revealed feature pairs with correlations exceeding 0.99. In an unconstrained MLP, the optimizer can assign arbitrarily large weights to redundant features and cancel them out — resulting in numerically unstable and non-generalizing weight configurations. L2 penalizes this directly by shrinking large weights toward (not to) zero.

**Hyperparameter tuning:** Tested `alpha` values of `{0.01, 0.1, 0.001}`. Selected `alpha=0.1` as the optimal value based on validation F1-Score. Higher values caused underfitting; lower values provided insufficient regularization.

#### 3. MLP + Early Stopping (Week 4, Part 5)

**Theoretical basis (Algorithm 7.1):** Early stopping monitors validation loss at regular intervals. When the validation loss fails to improve for `patience` consecutive checks, training halts and the model weights from the best checkpoint are restored:

```
if val_loss < best_val_loss:
    save checkpoint
    reset patience counter
else:
    increment patience counter
    if patience exceeded: stop training
```

**Connection to L2 (Week 4 theory):** Early stopping is geometrically equivalent to L2 regularization — it limits how far gradient descent can move parameters from their initialization (which is near zero), implicitly constraining weight magnitude.

**Hyperparameter tuning:** Set `patience=10` validation checks. This was chosen to allow the optimizer sufficient exploration time while preventing excessive memorization.

#### 4. Results

Both L2 and Early Stopping successfully recovered the Recall to match the Logistic Regression baseline (98.8%), confirming that the Plain MLP's performance drop was entirely attributable to overfitting rather than any fundamental architectural limitation.

| MLP + Early Stopping | 99.1% | 99.3% | 98.8% | 99.0% |

## Step 5: Optimization Analysis (Week 5)

#### 5a. Experimental Scope and Motivation

Step 4 established that regularization (L2 or Early Stopping) restores the model to the Logistic Regression performance ceiling. Step 5 asks the next natural question: given a well-regularized model, do the optimization dynamics themselves matter? This corresponds to Week 5 material covering adaptive optimizers, weight initialization theory, and learning rate scheduling.

All Step 5 experiments use PyTorch (replacing sklearn's MLPClassifier) for fine-grained control over initialization and learning rate schedules. The architecture (30→64→32→1, ReLU, Sigmoid output) and L2 alpha=0.1 are kept identical to Step 4 to isolate the optimization variables.

#### 5b. Optimizer Comparison (Week 5 — Adam vs SGD+Momentum)

**Three configurations tested:**

| Optimizer | Configuration | Theoretical Basis |
|---|---|---|
| Adam | lr=0.001, β₁=0.9, β₂=0.999 | Adaptive per-parameter LR using 1st + 2nd moment estimates with bias correction |
| SGD + Momentum | lr=0.01, momentum=0.9 | Exponential moving average of gradient direction |
| Adam + Cosine Annealing | Same Adam + ε_t = ε_min + 0.5(ε₀−ε_min)(1+cos(tπ/T)) | Combines adaptive LR with cyclic warm restarts |

**Connection to Step 1 multicollinearity:** The 30 features contain pairs with correlations up to 0.99. This produces gradients of very different magnitudes across parameters. Adam's per-parameter adaptive learning rate directly handles this heterogeneity — SGD applies the same step size everywhere, which is suboptimal when gradients are highly non-uniform.

**Output:** `outputs/figures/step6/optimizer_comparison_loss_curves.png`, `outputs/tables/step6_optimizer_comparison.csv`

#### 5c. Weight Initialization Comparison (Week 5)

**Four initialization strategies tested:**

| Initialization | Formula | Designed For |
|---|---|---|
| He (Kaiming) | W ~ N(0, 2/n_in) | ReLU networks — accounts for the ~50% dead activation rate |
| Xavier (Glorot) | W ~ N(0, 2/(n_in+n_out)) | Sigmoid/Tanh — equal variance across layers |
| Small Random | W ~ N(0, 0.01) | General baseline; risks vanishing gradients in deep networks |
| Large Random | W ~ N(0, 1.0) | Intentionally bad — demonstrates gradient explosion |

**Expected finding:** He initialization is theoretically optimal for this network (ReLU hidden layers). Xavier and Small Random provide comparison baselines. Large Random demonstrates pathological behavior, connecting to Week 3 theory about weight scale sensitivity.

**Output:** `outputs/figures/step6/initialization_comparison.png`, `outputs/tables/step6_initialization_comparison.csv`

#### 5d. Learning Rate Schedule Comparison (Week 5)

**Three schedules tested with Adam optimizer:**

| Schedule | Behavior |
|---|---|
| Constant LR | No decay — standard baseline |
| Step Decay | Halves LR every 30 epochs |
| Cosine Annealing | Smooth decay following cosine curve from ε₀ to ε_min |

**Why Cosine Annealing is preferred:** Step decay introduces abrupt LR changes that can destabilize training at the transition points. Cosine annealing provides smooth monotonic decay that allows the optimizer to settle more consistently in flat minima — a property particularly relevant for small datasets where overfitting risk is high.

**Output:** `outputs/figures/step6/lr_schedule_comparison.png`, `outputs/tables/step6_lr_schedule_comparison.csv`

#### 6. Why L1 and Data Augmentation Were Not Included

**L1 Regularization:** L1 induces sparsity by driving weights to exactly zero (constant gradient magnitude regardless of weight size). For this dataset, sparsity is not the goal — all 30 features carry meaningful diagnostic information. The multicollinearity problem is better addressed by L2 (which distributes weight across correlated features) than L1 (which eliminates some entirely). Adding L1 would broaden the project without adding methodological depth relevant to this specific dataset.

**Data Augmentation:** Data augmentation techniques (rotations, flips, crops) are designed for structured data with spatial invariances — primarily image data. This dataset consists of tabular numerical measurements; geometric transformations have no label-preserving interpretation here.

### Technical Decisions and Justifications

| Decision | Justification |
|---|---|
| L2 alpha=0.1 | Chosen by validation F1-Score sweep over {0.001, 0.01, 0.1}; lower values insufficient, higher values underfitted |
| Early stopping patience=10 | Conservative enough to allow exploration, strict enough to prevent memorization |
| Controlled experiment (one variable) | Isolates causal attribution; follows scientific methodology standard |
| No L1 regularization | Sparsity is not the goal; L2 is more appropriate for correlated continuous features |
| PyTorch for Step 5 | sklearn MLPClassifier does not expose per-layer initialization or custom LR schedulers; PyTorch provides the necessary control |
| He initialization as primary | ReLU networks require variance = 2/n_in to prevent gradient decay; theoretically motivated by ReLU's ~50% zero output rate |
| Cosine Annealing as primary schedule | Avoids abrupt step-decay transitions; smooth decay empirically produces more stable convergence on small datasets |
| Same architecture across all Step 5 experiments | Isolates optimization variable; architecture changes confound comparisons |
