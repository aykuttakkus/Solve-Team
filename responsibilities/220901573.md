### What I Did

#### 1. Methodological Justification: Why MLP After Logistic Regression

The transition from Logistic Regression to an MLP is the central architectural question of Week 3. A Logistic Regression model is a single-layer network — it can only produce a linear decision boundary. An MLP introduces hidden layers with nonlinear activations, allowing the model to learn nonlinear feature interactions.

**Theoretical motivation (Week 3, Part 1 — XOR Problem):** Even though the Breast Cancer dataset is largely linearly separable, the 30-feature space may contain nonlinear interaction effects that a linear model cannot capture. The MLP tests whether additional representational capacity yields measurable improvement.

**Key question being tested:** Does architectural complexity translate to better generalization, or does it introduce instability (overfitting)?

#### 2. Architecture Design

Selected a 2-hidden-layer architecture:

```
Input (30) → Hidden Layer 1 (64, ReLU) → Hidden Layer 2 (32, ReLU) → Output (1, Sigmoid)
```

**Activation function choice (Week 3, Part 2):**
- Hidden layers: **ReLU** (`max(0, z)`) — avoids the vanishing gradient problem present in sigmoid/tanh for intermediate layers; computationally efficient; standard choice for deep networks
- Output layer: **Sigmoid** — appropriate for binary classification, outputs a probability in [0, 1]
- **Why not Softmax?** Softmax is for multiclass problems (K > 2). Binary classification is handled more efficiently with a single sigmoid output unit and Binary Cross-Entropy loss

**Why not deeper?** The dataset has 569 samples. Deeper architectures (3+ hidden layers) would dramatically increase the risk of overfitting given the small sample size, without theoretical justification for this feature space.

#### 3. Training Configuration

- **Loss function:** Binary Cross-Entropy
- **Optimizer:** Adam (adaptive learning rate)
- **Batch training** via sklearn's `MLPClassifier` with loss curve tracking enabled
- Same stratified splits from Step 1, applied identically

#### 4. Overfitting Analysis

The primary contribution of this step is visualizing the overfitting gap. Training and validation loss curves are plotted across epochs to reveal the divergence point where the model begins memorizing training samples rather than learning generalizable patterns.

**Theoretical connection (Week 3 → Week 4 bridge):** When training loss continues to decrease while validation loss plateaus or increases, the model has exceeded its optimal capacity for this dataset. This divergence is the explicit motivation for Week 4 regularization techniques.

#### 5. Results

The Plain MLP showed slightly lower performance than Logistic Regression on the validation set, with visible overfitting in the loss curves. This is a meaningful technical finding — not a failure — because it demonstrates that architectural complexity does not guarantee better generalization, and directly motivates the regularization experiments in Step 4.

### Technical Decisions and Justifications

| Decision | Justification |
|---|---|
| Architecture: 30→64→32→1 | Small enough to avoid catastrophic overfitting, large enough to demonstrate the gap; follows Week 3 "depth vs. width" guidance |
| ReLU in hidden layers | Avoids vanishing gradient; standard for tabular classification; directly covered in Week 3 Part 2 |
| Sigmoid output | Binary classification requires single-unit probability output; Softmax would be redundant |
| BCE loss | Convex loss for logistic outputs; stronger gradients when model is wrong vs. MSE with sigmoid |
